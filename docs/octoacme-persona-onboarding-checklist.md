# OctoAcme Persona Onboarding Checklist

Use this checklist for quickly onboarding new team members in each persona/role and making sure they understand their responsibilities and collaboration points.

## General steps for all personas
- [ ] Read OctoAcme Project Management Overview and relevant role docs
- [ ] Participate in intro meeting with PM and PdM
- [ ] Review key templates: One-pager, risk register, role matrix
- [ ] Add yourself to communication channels (standup, syncs, etc.)

## Persona-specific onboarding

### Scrum Master
- [ ] Review current sprint artifacts and team routines
- [ ] Schedule/facilitate ceremonies: standup, review, retro
- [ ] Connect with PM for process gaps and improvements

### UX/UI Designer
- [ ] Review existing backlog items with visual/design requirements
- [ ] Coordinate with PdM and Developer for handoff
- [ ] Assess accessibility standards in current release

### Technical Writer
- [ ] Clone latest docs and engineering references
- [ ] Meet Developers and QA for doc pain points
- [ ] Review doc publishing templates and process

### QA Automation Engineer
- [ ] Review automated test pipeline in CI
- [ ] Sync with Developer and QA for coverage gaps
- [ ] Document test case review process

### DevOps Engineer
- [ ] Audit CI/CD pipeline setup and deployment notes
- [ ] Confirm incident playbook access
- [ ] Document monitoring and rollback procedures
